# 🏗 **EquiManage**

## 📌 **Project Overview**
**EquiManage** is a **console-based basic project** designed to help users **buy, sell, and manage equipment** efficiently. The system ensures seamless transactions and maintains records of available equipment.

### 🚀 **Key Features**
- **Buyer & Seller System:** Separate interfaces for buyers and sellers.
- **Equipment Management:** Allows sellers to add equipment and buyers to view & purchase items.
- **Authentication:** Simple login mechanism for buyers and sellers.
- **Token System:** Uses tokens to manage equipment transactions securely.

---

## 🏗 **Project Structure**
- **`EquipmentManagement.java`** - The main class handling the overall flow of the system.
- **`Buyer.java`** - Manages buyer registrations, logins, and interactions.
- **`Seller.java`** - Handles seller logins and inventory management.
- **`Equipments.java`** - Maintains the list of available equipment and their details.
- **`Token.java`** - Generates unique transaction tokens.

---

## 🛠️ **Setup & Execution**
1. **Clone the repository**
   ```sh
   git clone https://github.com/B-Vighnesh/EquiManage.git
   cd EquiManage
   ```
2. **Compile and Run**
   ```sh
   javac EquipmentManagement.java
   java EquipmentManagement
   ```
3. **Follow the console prompts** to login as a **buyer or seller** and interact with the system.

---

## 🔧 **Use Cases**
EquiManage can be used in various fields where equipment management is required:
- **Electronics Labs:** Manage lab equipment availability and ensure proper tracking of borrowed tools.
- **Construction Sites:** Keep track of heavy machinery and tools rented out to different contractors.
- **University Labs:** Help students and staff manage lab equipment usage efficiently.
- **Event Management:** Maintain records of rented equipment for events and conferences.

---

## 🔐 **Security Considerations**
- This is a **basic console-based project**, so it does not use a database or advanced authentication.
- The project currently **stores credentials in plain text**, which can be a security risk.
- Consider implementing **hashed passwords** and **database storage** for enhanced security.
